The Full Picture

Every dimension where consolidation onto Stripe Connect changes the equation for Custom Ink.

⚠️ Today — Three Silos
🏗️

Three separate platforms

GOF, Fundraising, and Stores each have their own system, codebase, and ops runbook. No shared infrastructure.

💸

5+ payout destinations, different cadences

Fundraising: Tipalti checks (weekly) + PayPal Tuesday runs. Stores: legacy PayPal mass-pay (monthly, $60K/batch cap) + Tipalti ACH Net60 migration in progress. GOF: Braintree + manual wires.

📋

Compliance debt blocking payouts

Fulfill Engine stores can't legally distribute profits yet — 1099/W-9 collection, IRS validation, escheatment tracking, and cross-system COGS recognition all unresolved. Payout flow is manual end-to-end: STS → Finance → Tipalti approval chain.

🔒

W-9 collection per payee, every new store

New Tipalti payees must register via email link, submit W-9, and select ACH before receiving any payment. Unregistered email = blocked payout. No shared identity layer across flows.

🛡️

Fraud siloed per platform

A fraudster blocked on fundraising can still transact on stores. No cross-platform Radar rule enforcement.

🔧

GOF "mega complicated"

Described internally as over-engineered for its business value. Technical debt without a clear remediation path.

✨ With Stripe Connect
⚡

One platform, three configurations

Stripe Connect is the shared layer. Each flow uses different Connect capabilities (destination charges, Express accounts, Separate Transfers) on the same API.

🏦

One payout API

stripe.payouts.create handles everything — org disbursements, store payouts, fulfillment partner splits. One contract, one support team, one API key.

📊

One Sigma report

SELECT * from all three flows in one query. Cross-flow revenue analytics, platform fee P&L, and donor LTV without a data warehouse.

✅

KYC once, works everywhere

Connected account created once per org. That account works for fundraising campaigns, online stores, and future products Custom Ink builds on Connect.

🤖

Radar across all flows

Rules configured once apply everywhere — velocity, geo-risk, card-testing detection. A fraud signal on one flow informs the others.

🧹

GOF integrated into core

GOF absorbed into Custom Ink's core payment process — Scott's stated goal. Stripe Connect is the "core payment process" it integrates into.

①

Org Onboards (One-Time)

Nonprofit clicks "Create Fundraiser" on Custom Ink. Stripe Connect Onboarding opens in-flow — collects EIN, officer info, bank account (verified via Financial Connections). Custom Ink's backend stores the Stripe Account ID. Org is now a connected account.

POST /v1/accounts → GET account_link → org completes KYC → webhook: account.updated (transfers_enabled: true)
②

Custom Ink Sets Campaign Parameters

Custom Ink platform team configures: product pricing, campaign duration, whether standalone donations are enabled, payout schedule (manual at campaign close), and whether platform takes an application fee (0% for nonprofits, 2–5% for groups).

③

Donors Purchase & Give

Donor loads the campaign page, selects products, optionally adds a donation. Stripe Payment Element shows Apple Pay, Google Pay, Link, cards. PaymentIntent is created with on_behalf_of set to the org's connected account ID.

POST /v1/payment_intents { on_behalf_of: acct_xyz, transfer_data: { destination: acct_xyz, amount: donation_cents } }
④

Stripe Routes Funds & Runs Fraud Checks

Radar scores every transaction in real-time. Destination charge sends product margin to Custom Ink's Stripe account. Donation transfer routes to org's connected account balance — where it sits until payout is triggered.

Radar ML score → allow/block/3DS → funds split: $24 stays on platform, $10 transferred to acct_xyz
⑤

After all orders are printed and shipped, Custom Ink's backend calls stripe.payouts.create on behalf of the connected account. Org receives funds via ACH in 1–2 days or instantly if enabled. Org sees payout status in their embedded dashboard.

POST /v1/payouts { amount: balance_available } + stripeAccount: acct_xyz header → payout.paid webhook → notify org

Why Migrate from Hyperwallet → Stripe Connect?

Hyperwallet (PayPal Advanced Payouts) is purpose-built for outbound mass payouts to gig workers — but requires a separate payment processor for donor-facing checkout. Stripe Connect collapses the entire stack: checkout, routing, fraud, payouts, and tax reporting all live in one API. This eliminates reconciliation between two systems, gives donors a better checkout experience (Apple Pay, Link), shows the org's name on bank statements, and gives Custom Ink a single data model for analytics.

What Donors Gain

One-click Apple Pay / Google Pay checkout. Org name on their bank statement. Instant receipt with EIN. Link saves their card for future campaigns. Better fraud protection means fewer false declines.

What Orgs Gain

15-minute onboarding with no paper forms. Real-time dashboard embedded in Custom Ink portal. Instant Payouts option. Automated tax documents. No need to manage a PayPal account.

What Custom Ink Gains

Single platform for both checkout and payouts — one API, one data model, one support team. Programmatic payout triggers aligned to order fulfillment. Radar fraud protection across all campaigns. Sigma for cross-platform analytics. The ability to offer recurring monthly donations — a revenue and retention lever unavailable with Hyperwallet alone.
